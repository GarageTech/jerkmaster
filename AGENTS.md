# Project Instructions

- Update `CHANGELOG.md` for every functional, hardware, interface, documentation, or safety change.
- Include the changelog update in the same commit as the related change.
- Add new work to the `Unreleased` section and keep historical commit entries in chronological order.
- Do not include generated previews or files from `tmp/` in commits.
- Keep the deployable JerkMaster web service Raspberry-only; do not add a second local-computer server path.
